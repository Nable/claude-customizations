# Brand Identity Generator

A Claude Code custom command that analyses an app project and generates a complete brand identity document: brand foundations, visual identity (palette, typography, icon), tone of voice, key messages, personas, and usage guidelines.

---

## How it works

```
Claude Code (inside the project)
──────────────────────────────────────────────────
Reads app-store-listing.md, marketing.md, README
Extracts colours from UI code (SwiftUI, Android XML)
Extracts fonts declared in the project
Analyses UI strings for tone of voice
         ↓
Generates brand-identity.md
```

Where possible, the document is built from real project data (colours, fonts, existing copy) rather than generic templates. Every value extracted from code is explicitly marked.

---

## What `brand-identity.md` contains

| Section | Content |
|---------|---------|
| **Foundations** | Mission, vision, values, brand personality |
| **Tone of voice** | Profile, spectrum across 4 axes, do's & don'ts, examples for each UI message type |
| **Colour palette** | Primary, secondary, neutral, semantic — with HEX and RGB; WCAG contrast check; dark mode notes |
| **Typography** | Primary and secondary fonts, full hierarchy with weights and sizes |
| **App icon** | Description of detected visual characteristics + guidelines |
| **Visual style** | Iconography, illustrations, spacing, border radius, elevation |
| **Key messages** | UVP, tagline, elevator pitch, messages calibrated per channel |
| **Personas** | 2–3 user profiles derived from real category and features |
| **Digital presence** | Recommended social handles, domains, email addresses by role |
| **Usage guidelines** | Rules for logo, colours, and typography |
| **Checklist** | Control list to complete the identity |

## Installation

### Prerequisites

- [Claude Code](https://claude.ai/code) installed and configured

### Install the custom command

Create the custom commands folder if it doesn't exist:

```bash
mkdir -p ~/.claude/commands
```

Copy the command file:

```bash
cp brand-identity.md ~/.claude/commands/brand-identity.md
```

The command will be available in Claude Code as `/project:brand-identity`.

---

## Usage

Open the project in Claude Code and run:

```
/project:brand-identity
```

### Recommended workflow

Run the commands in order — each one feeds the next:

```
/project:app-store-listing   →   /project:marketing-advisor   →   /project:brand-identity
```

The Brand Identity Generator reads files generated by the previous commands to avoid redundant analysis and keep all documents consistent.

---

## Notes

- Colours extracted from code (SwiftUI `Color`, `UIColor`, Android `colors.xml`) take priority over recommendations and are reported with the *(from code)* marker.
- Tone of voice is derived from existing UI strings in the project: if the current copy is formal, the document reflects that and flags any changes as intentional decisions.
- The file is designed to be shared with designers and copywriters as a single source of truth for the brand identity.
